# 競合調査・比較表作成スキル

市場・競合を体系的に調査し、意思決定に使える比較表・レポートを作るスキル。

## まず確認すること

1. **調査対象のサービス/事業** — 何と競合を比べるか
2. **調査の目的** — 参入判断 / 差別化戦略 / 価格設定 / 機能比較など
3. **比較軸の優先度** — 機能・価格・ターゲット・UI/UXなど
4. **出力形式** — 比較表 / レポート / スライド資料
5. **調査範囲** — 国内のみ / グローバルを含むか

---

## 調査の進め方

### Phase 1: 競合の洗い出し

以下の観点で競合を分類する：

**直接競合**（同じ課題を同じ手段で解決）
- 全く同じターゲット・機能・提供価値を持つサービス

**間接競合**（同じ課題を違う手段で解決）
- Excelや手作業など、アナログで同じ課題を解決している手段も含む

**代替手段**（ユーザーが今やっていること）
- 競合サービスではないが、同じニーズを満たしている行動

> 「競合がいない」は市場がないか、まだ気づいていないかのどちらか。必ず代替手段まで含めて考える。

### Phase 2: 比較軸を決める

目的に応じて比較軸を選ぶ。以下は代表的な軸：

**機能・サービス面**
- コア機能の有無
- 独自機能・差別化ポイント
- 対応OS・プラットフォーム
- 連携サービス

**ビジネス・価格面**
- 料金体系（月額/従量/買い切り）
- 無料プランの有無
- 最低契約期間

**ターゲット・市場面**
- ターゲット顧客（業種・規模・職種）
- 主要ユーザー層
- 対応言語・対象地域

**体験・品質面**
- UI/UXの印象
- サポート体制（チャット/電話/FAQ）
- ドキュメントの充実度
- 導入実績・事例数

### Phase 3: 情報収集

各競合について以下のソースから情報を取得する：

1. **公式サイト** — ランディングページ・料金ページ・機能一覧
2. **レビューサイト** — G2 / Capterra / 比較.com / ITreview
3. **App Store / Google Play** — アプリのレビュー・評価
4. **SNS** — Twitter/X でのユーザーの声・不満
5. **求人情報** — 採用ページからエンジニアリング・事業規模を推測
6. **PR TIMES / プレスリリース** — 資金調達・新機能情報

## ポジショニングの考え方

競合調査の最終目的は「自社がどこで戦うか」の判断。以下のフレームで整理する：

**ブルーオーシャンを探す**
- 競合が揃って弱いセグメント
- 誰も取っていない比較軸（例：「〇〇業界特化」）

**競合が強い場所では戦わない**
- 後発で追いかけてもコストが嵩むだけ
- 「自社が最も差別化できる軸」にフォーカスする

**ユーザーの不満から逆算する**
- レビューの「不満点」は機会を示している
- 競合が対応できていないニーズは自社が取りにいく場所

---

## 調査の注意点

- **公式サイトの情報は楽観的**：実際の使い勝手はユーザーレビューで確認する
- **価格は変動する**：定期的に再確認が必要
- **「競合なし」は危険シグナル**：本当にないのか、見つけられていないのかを区別する
- **スクリーンショットや日付を記録する**：競合は変化が速いので時点を残す
- **自社視点だけで評価しない**：競合の強みを過小評価しない
